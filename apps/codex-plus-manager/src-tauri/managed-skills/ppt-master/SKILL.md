# PPT Master

Create and improve presentation decks with executive-level structure, clear storyline, and slide-ready content.

## Workflow

1. Clarify the audience, goal, time limit, language, and whether the deck should be formal, sales-oriented, training-oriented, or strategy-oriented.
2. Build a storyline before writing slides: context, core problem, insight, solution, evidence, action.
3. Convert paragraphs into slide-ready bullets. Each slide should have one main message and concise supporting points.
4. Use strong slide titles that state the conclusion, not generic labels.
5. When creating a deck, provide a slide outline with title, purpose, key content, and suggested visual layout.
6. When editing a deck, flag weak logic, crowded slides, missing evidence, and repeated ideas.

## Quality Rules

- Avoid long paragraphs on slides.
- Keep one primary idea per slide.
- Use charts, tables, timelines, process diagrams, or comparison layouts when they make the point clearer.
- Do not fabricate data. Mark missing evidence clearly.

## Output

Return slide-ready content or a clear deck outline. If the user asks for a file, create or modify the actual deck using the available local tooling.
